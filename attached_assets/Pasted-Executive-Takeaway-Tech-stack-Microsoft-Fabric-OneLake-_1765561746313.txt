Agent Strategy
1. Core Agent Types
CareGuide Agent: Conversational intake assistant for families; validates coordinator entries; generates outcome metrics.
Compliance Agent: Automates HUD/CMS filings, assembles evidence, enforces HIPAA/PII rules.
Document Agent: Drafts donor letters, grant narratives, board reports, and audit bundles.
Engagement Agent: Prioritizes donors, volunteers, and funders; prevents cannibalization; schedules outreach.
Financial Agent: Consolidates P&L, detects double counting, forecasts cash flow, tracks donor acquisition costs.
Orchestration Agent: Coordinates tasks across modules, enforces SLAs, escalates anomalies.
Training Agent: Provides contextual help, micro-learning, and adoption nudges inside workflows.
2. Agent Operating Principles
Embedded in modules: Each agent lives inside its relevant app hub (e.g., CareGuide Agent inside CareGuide 2.0).
Human-in-the-loop: Sensitive actions (filings, donor comms, merges) require staff approval.
Audit trails: Every agent action logged for compliance and trust.
Feedback loops: Staff can correct or rate AI outputs; agents learn continuously.
Event-driven: Agents respond to triggers from Azure Event Grid (e.g., intake submitted, donor pledge logged, compliance deadline approaching).
